Root Certificates

If you are experiencing connection issues, it may be due to outdated root certificates in your browser.

By default, PolyMSNP is configured to connect to your selected service via HTTPS, and outdated certificates can cause login to fail. While it is possible to force HTTP connections, it is not recommended as it can expose your credentials to attackers and is only provided for compatibility reasons.

Resolving the issue depends on your operating system, although in most cases as long as you are using an up-to-date operating system, your root certificates should be updated automatically.

  • Windows XP and later
    A tool exists on GitHub to update root certificates called UpdateRootCertificates. You can download the latest release from here.
  • Mac OS X, iPhone OS, iPad OS
    DER-encoded root certificates can be downloaded from here, with the option of downloading individual certificates or a .mobileconfig bundle. Do note that:
    • .mobileconfig bundles will require at least Mac OS X 10.7.0 or iOS 3.0. Signed .mobileconfig bundles will require at least Mac OS X 10.7.3 or iOS 5.0.
  • Android
    The installation method depends on what Android version you are using.
    • Android 7.0 and later: DER-encoded root certificates can be downloaded from here. After downloading, you can tap on the file to install it.
    • Android 6.0 and earlier: You will need to convert the DER-encoded certificate to PEM format, then copy it to your device and install it through the security settings. You can use OpenSSL to convert the certificate with the following command: 
      openssl x509 -inform der -in certificate.cer -out certificate.pem
